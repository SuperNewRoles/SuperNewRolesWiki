役職提案について

提案方法

提案規約

  • 適切な場所で提案を行ってください。
  • 一つのチケットにつき一つの役を提案してください。
  • 最低評価基準を満たす内容で提案を行ってください。
  • 基準を満たしていない場合、「規約を読んでいない」と判断し、返答せずクローズさせて頂く事もあります。

提案場所

  • SNR公式Discordサーバのこちらのボタンからチケットを作成してください。
  • 作成されたチケットで提案を行ってください。

  • チケットで提案した例

  • 一つのチケットにつき一つの役を提案してください。

最低評価基準

共通基準

  • コンセプトブレはないか
  • コンセプトブレの例
    • 名前と能力が一致していない
    • 能力を詰め込みすぎで一貫性がない
    • 以下の問に返答ができない
    • 「その役職で何がしたいか」
    • 「その役職を入れる事で何が起きるか」
    • 「その役職がゲームにどのような影響を与える事を期待しているか」
    • 「既存役に似た役職があるが, どこを差別化したか」
    • 「その役職を入れる意味があるか」
    • etc...

  • 他者, 他陣営の行動に干渉しすぎていないか。
  • 過剰に妨害していないか
  • ストレスを与えすぎてないか

  • 他のModの役職をそのまま提案していないか。
  • 他Modの役をSHR対応して欲しい等で, 他のModの役職を提案したい場合は要望で行ってください
    • 対応する可能性は限りなく低いです。
    • [悪女]の提案は禁止しています。

  • 死体を冒涜する行為ではないか

インポスター役職

  • バランスブレーカーでないか。
  • ゲーム性を壊していないか。
  • ゲーム性を壊す例
    • 推理要素を与えないキル能力
    • 回数制限がある及びデメリットがあればここは考慮の余地あり
    • 推理要素を与えない過剰なクルー妨害能力
    • 一人で勝利が完結する
    • アクションゲームにする

第三陣営役職

  • ゲーム性を壊していないか。
  • ゲーム性を壊す例
    • インポスター及びクルーに対処できる情報と能力を渡さず, 一人で勝利が完結する
    • インポスターVSクルーの構図を過剰に壊す
    • 自分以外をアクションゲームにする能力

  • バランスブレーカーでないか。
  • 第三陣営は試合の調整役として用いる場合もある為、バランスやゲーム性を壊す物であったとして使用意図(*1)を明確に説明できる場合考慮する可能性がある
*1 : [使用意図テンプレート]
- [制御したい問題点]  - - [何故制御したいか]  - - [役職によってどのように制御できるか]  -
使用例 (てるてる)
  • [制御したい問題点]
  • ローラー、詰めが甘い吊り
  • [何故制御したいか]
  • 本来情報を精査すればローラー位置を狭められるはずなのに詰めずに決行してしまうのは問題があるから。
  • 現行犯によるローラーであってもそれまでの情報を考慮すれば単吊り出来る可能性があるのにも関わらず、精査せず決行してしまうのに問題があるから。
  • [役職によってどのように制御できるか]
  • ローラー位置にてるてるがいる可能性を考慮する必要が生じる事で、吊りに対して慎重になり、情報精査を細かく行うようになると考えられる

クルーメイト役職

  • バランスブレーカーでないか。

  • ゲーム性を壊していないか。
  • ゲーム性を壊す例
    • 乗っ取りができない確定白をつくることができる
    • 一人で勝利が完結する
    • インポスターの行動を過剰に制限する
    • アクションゲームにする

補遺

  • 役職カラー
  • [~系], 又は[RGB]で指定してください。
    • [RGB]で指定した場合, 他役職との兼ね合い等で, 色を系統を維持しつつ変更する場合があります。その旨をご了承願います。
  • 例
    • 警察医

    • 水色系の色
      又は
      R:137, G:195, B:235

  • パッシブ能力
  • 常時発動する能力。
  • 例
    • スター
    • カリスマ, 溢れ出る輝き

  • アクティブ能力
  • トリガーを踏む事で発動する能力
    • トリガー例
    • ボダン動作
    • タスク完了
    • 追放

  • 能力例
    • 黒猫
    • タスクトリガー : 狂信
    • 追放トリガー : 道連れ

  • 設定項目の表記法
  • 数値で設定する物
    • (初期値:, 最低値:, 最大値:, 間隔:)
    • 例
    • バイタル画面クールダウン
      • (初期値 : 15秒, 最低値 : 5秒, 最大値 : 60秒, 間隔 : 5秒)

  • On/Offで設定する物
    • 初期値:[On Offのいずれかを記載]
    • 例
    • タスクフェイズ中に携帯バイタルを所持しているか。
      • 初期値 : OFF

使用例

[役職名]

  • ～ 警察医 / Police Surgeon ～

[イントロ]

  • 私は幸せな最期を看取る手伝いをしたかっただけなのに...

[陣営]

  • クルーメイト

[役職カラー]

  • 水色系
  • R : 137, G : 195, B : 235

[コンセプト]

  • SHRでは科学者が使えない事からバイタルを見れる役職が無い為追加したい

  • SNRドクターのSHR移植
  • SNRドクターはSHRでは動かせない為「ドクターSHR対応」ではなく、似た別役職が必要だと考えた
  • SNRドクター由来の為、能力の軸は死亡時刻の確認とする

[Description]

  • 会議中、死亡時刻を確認する事ができる役職です (設定により死亡推定時刻に変更可能です)。
  • 設定によりタスクフェイズ中に携帯バイタルを使用する事もできます。

[説明]

  • 能力1_アビリティ：携帯バイタル
  • 能力の元ネタ
    • 死亡届に必要な、死亡をしている事を証明し、死因が事故、自殺、突然死、不明の時の公的文書。
    • https://www.mhlw.go.jp/toukei/manual/
    • https://www.mhlw.go.jp/toukei/manual/dl/manual_r05.pdf
  • 携帯バイタルの所持は設定可能。

  • バニラ役職 : 科学者が有する能力 (以下バニラ能力と同様の説明)
    • バイタル画面クールダウン
    • タスクがすべて完了後に, 充電持続時間が切れた後に再び使用可能になるまでのクールダウン

    • 充電持続時間
    • バイタルが使用可能な残時間。
    • タスクが一つ完了する毎に回復する。
    • タスクが全て終了した後は、クールダウン経過後に回復する。

  • 能力2_パッシブ : 死体検案
  • 会議開始時に死体検案書をチャットに送信する(通常モード, SHRモード共通)。

    • 検案書の再送(設定)
    • 通常モード
      • 全文の送信(チャット)
      • 自分自身のネームプレートに表示されているボタンを押す。
      • 個人の検案書の表示
      • 確認したい人のネームプレートに表示されているボタンを押す。

    • SHRモード
    • 全文の送信(チャット)
      • 自分自身に投票する。
      • この設定が有効時, 警察医は自投票不可。

    • 死亡したターンの会議フェイズ終了以降も死亡秒数を記載するかは、設定する事ができる。
    • 死因表記で、キルによる死亡か追放による死亡かが確認可能。
    • 上記詳細は, 検案書の内容に記載。

    • 何ターン前の死亡かを記載するかは、設定する事ができる。

    • 死亡時刻に誤差を含めることができる(死亡推定時刻) (設定)
    • 設定有効時
      • 誤差の範囲秒数を設定する。
      • 例 : 5秒で設定した場合
      • (死亡時刻 - 5秒) ~ (死亡時刻 + 5秒)の 範囲で死亡推定時刻をランダムで決定する。
      • 死亡推定時刻を記載し、「 (推定)」と表記する。

    • 設定無効時
      • 死亡時刻を記載し、「(確認)」と表記する。

    • 検案書の内容
    • 死亡してから、そのターンの会議開始までの経過時間(死亡(推定)時刻)

    • 死亡(推定)時刻を記載できない場合
      • 秒数ではなく「(頃)」と表記する。
      • 例
      • スラッガーキル等死体が出ないキル。
      • [死亡したターンの会議フェイズ終了後も、死亡秒数を表記するか]有効時の会議中キル。
      • [死亡したターンの会議フェイズ終了後も、死亡秒数を表記するか]有効時の追放による死亡。

    • 死因
      • 通常の追放により死亡した場合
      • 死因 追放

      • 通常追放以外
      • 死因 不詳の外因

[設定]

  • タスクフェイズ中に携帯バイタルを所持しているか。
  • 初期値 : OFF
    • バイタル画面クールダウン
    • (初期値 : 15秒, 最低値 : 5秒, 最大値 : 60秒, 間隔 : 5秒)

    • 充電持続時間
    • (初期値 : 5秒, 最低値 : 5秒, 最大値 : 30秒, 間隔 : 5秒)

  • 死体検案書の再送を可能にするか。
    • 初期値 : OFF

  • 死亡したターンの会議フェイズ終了後も、死亡時刻を表記するか。
  • 初期値 : ON
    • 会議フェイズ中その死者が何ターン前に死亡したかを表示するか。
    • 初期値 : OFF

  • 中華民国歴表記にするか
    • 初期値 : ON

  • 死亡時刻に誤差を含めるか。
    • 初期値 : ON

  • 死亡時刻に含める誤差の範囲(絶対値)。
    • (初期値 : 5秒, 最低値 : 1秒, 最大値 : 15秒, 間隔 : 1秒)
